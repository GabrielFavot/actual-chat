  <action>
Enhance the poll function to format and send notifications properly.

**Notification formatting:**
- Reuse existing transaction formatting from `src/commands/transaction.ts`
- Build inline keyboard with category buttons (same as /transaction command)
- Send via `bot.api.sendMessage(userId, message, {reply_markup: keyboard})`
- Store transaction in sessionManager for callback handling

**Transaction formatting (from existing /transaction command):**
1. Get newest uncategorized transaction
2. Fetch accounts and payees (use existing API methods)
3. Get account name, payee name, amount, date, category (if any)
4. Format message with HTML: "📅 {date} | {account} | {payee} | {amount}"
5. Build category buttons inline keyboard (4 buttons per row max)
6. Store transaction in sessionManager for callback handling:
   - const sessionId = sessionManager.storeTransaction(transaction);
   - Build callback data: cat_{sessionId}_{categoryId}
7. Mark transaction as notified in NotifierState after successful send

**Example formatting (reuse from transaction.ts):**
```
📅 2026-03-12 | Checking | Costco | $125.50

[Groceries] [Gas] [Dining] [Shopping]
[Utilities] [Health] [Entertainment] [Other]
```

**Error handling in poll:**
1. Wrap entire poll logic in try-catch
2. Catch specific errors:
   - Network errors: log "Network error during poll: {error}"
   - API errors: log "API error during poll: {error}"
   - Transaction store errors: log but don't crash
3. Catch-all: log error, continue polling (don't throw, don't exit)
4. Success path: log "Poll {time}: found uncategorized, sent notification" or "Poll {time}: no new transactions"

**Mutex implementation:**
- Simple boolean flag at module level: `let isPolling = false`
- In poll function:
  ```
  if (isPolling) {
    console.log('Poll already in progress, skipping cycle');
    return;
  }
  isPolling = true;
  try {
    // ... poll logic
  } finally {
    isPolling = false;
  }
  ```

**Startup poll:**
- Call poll function immediately after `startPolling()` is called
- Don't wait for first interval - check right away
- Then schedule interval polling every 4 hours

**No transactions case:**
- Silently skip (don't send message)
- Log: "Poll complete: 0 uncategorized transactions found"

**Why this approach:** Consistent with Phase 3 UX, simple error handling, prevents crashed polling loop
