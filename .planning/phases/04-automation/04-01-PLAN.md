<task type="auto">
  <name>Task 2: Create PollScheduler for 4-hour polling with mutex and startup check</name>
  <files>src/utils/poll-scheduler.ts</files>
  <action>
Create PollScheduler utility that:
1. Polls every 4 hours for uncategorized transactions
2. Prevents concurrent polls (mutex/lock mechanism)
3. Runs startup check immediately
4. Sends notifications only if new uncategorized transactions exist

**Core function:**
```
startPolling(bot: Bot, actualApi: ActualApiService, authorizedUserId: number, notifierState: NotifierState)
```

**Poll logic:**
1. Call `actualApi.getUncategorizedTransactions()`
2. Get newest transaction: sort by created date (descending), take first one
3. Check: `notifierState.hasBeenNotified(newestTxn.id)`
4. If NOT notified:
   - Mark as notified: `notifierState.markNotified(newestTxn.id)`
   - Send notification: use existing `/transaction` formatting (from transaction.ts)
   - Log: "Poll complete: sent notification for {payee} {amount}"
5. If already notified OR no transactions:
   - Silent (log poll completion, don't send notification)

**Mutex implementation:**
- Use simple boolean flag `isPolling = false`
- Before poll: check if `isPolling`, if true skip this cycle (don't wait, just skip)
- After poll completes: reset flag

**Startup check:**
- On `startPolling()` call, immediately run poll once (don't wait for 4-hour interval)
- Then start interval timer for every 4 hours

**Error handling:**
- Catch errors during poll
- Log error with timestamp
- Don't crash - continue polling
- Don't send notification on error

**Hardcoded values:**
- Poll interval: 4 hours = 4 * 60 * 60 * 1000 ms = 14400000 ms
- No configurable settings for now

**Why this approach:** Simple event-driven polling, works in Docker, minimal dependencies, works with grammY bot object
