## Tasks

### Task 1: Create transaction command handler

**Goal:** Fetch and display first uncategorized transaction with category buttons

**Location:** Create `src/commands/transaction.ts`

**Specification:**

```typescript
/**
 * Handle /transaction command
 * Fetch first uncategorized transaction and display with category buttons
 */
export async function handleTransactionCommand(ctx: Context): Promise<void>
```

**Implementation steps:**

1. Command handler function:
   ```typescript
   bot.command('transaction', handleTransactionCommand);
   ```

2. Get uncategorized transactions:
   ```typescript
   const transactions = await actualApi.getUncategorizedTransactions();
   
   if (transactions.length === 0) {
     await ctx.reply('✓ All transactions are categorized!');
     return;
   }
   ```

3. Get first transaction and categories:
   ```typescript
   const transaction = transactions[0];
   const categories = await actualApi.getCategories();
   ```

4. Format message:
   ```
   📝 Transaction Details
   
   Date: 2026-03-12
   Payee: Coffee Shop
   Amount: -$5.50
   Account: Checking
   
   Select a category:
   ```

5. Build inline keyboard with categories:
   ```typescript
   const keyboard = new InlineKeyboard();
   
   categories.forEach(cat => {
     keyboard.text(cat.name, `cat_${cat.id}_${transaction.id}`);
   });
   ```

6. Send message with keyboard:
   ```typescript
   await ctx.reply(message, { reply_markup: keyboard });
   ```

**Error handling:**
- Catch API errors, show: "Error fetching transactions. Check logs."
- Handle no categories: "No categories available"
- Show user-friendly messages

## Callback Data Format

Button callback data: `cat_{categoryId}_{transactionId}`

Example: `cat_123e4567_89ab_4567_89ab_123456789abc`

⚠️ **Note:** Telegram limits callback_data to ~64 chars. Need to verify this works with UUID transaction IDs.

If IDs too long:
- Option A: Use shorter keys, store transaction context in memory
